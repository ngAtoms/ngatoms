Components / Actions

Button

A touchable, accessible action primitive. Buttons trigger an immediate effect — submit a form, open a dialog, advance a flow. Everything else should be a link.

ngAtoms v0.3+ ngatoms add button Stable Source

Usage

Add the button to your project and import the standalone directive wherever you need it. ngAtoms is tree-shakeable — only the components you add are bundled.

$ ngatoms add button

Then import NgAtomsButtonDirective into the component that uses it:

Variants

Five visual weights. Use primary for the single most important action on a surface, secondary or ghost for the rest, outline for tonal emphasis without a fill, and destructive for confirmed destructive actions.

Sizes

Five sizes. md is the default and covers most cases. Reserve lg / xl for hero CTAs and xs / sm for toolbars and dense areas.

With icon

Any content is allowed inside the button. Place an icon before or after the label — the button handles spacing and alignment. For icon-only buttons, always add an aria-label.

Loading

Pass [loading]="true" while an async action is in flight. The button keeps its width, disables pointer events, and shows a spinner. Screen readers announce the state via aria-busy.

Disabled

Disabled buttons don't trigger and are removed from the tab order. Prefer showing a tooltip that explains why a button is disabled rather than leaving the user guessing.

As link

When the action navigates, use an anchor tag styled with the button directive. It keeps the visual affordance but is a real link — right-click, middle-click, and keyboard navigation all behave correctly.

Accessibility

Built on a native <button> element, so most accessibility is handled by the browser. What ngAtoms adds on top:

@for (item of a11yItems; track item.title) {
{{ item.title }}
{{ item.body }}
}

Recipes

Submit + async

Bind [loading] to a signal so the button reflects in-flight state. This pattern works equally well with RxJS, NgRx, or plain promises.

Submit + error handling

Bind a second signal for the error state and flip the variant on failure. The button communicates failure without leaving the user on a broken-looking form.

Destructive confirmation

Pair a destructive button with a dialog so the user explicitly confirms before anything irreversible happens.
