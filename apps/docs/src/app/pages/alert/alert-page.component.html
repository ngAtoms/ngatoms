Alert

A callout banner for feedback, status, or system messages. Five semantic variants — default, info, success, warning, and destructive.

ngAtoms v0.3+ ngatoms add alert Stable Source

New version available

Version 2.0 includes breaking changes. Review the migration guide.

Usage

Add the alert to your project and import the standalone directives wherever you need them.

$ ngatoms add alert

Then import the directives into your component:

Variants

Five semantic variants for different message types. Use default for general notices, info for informational messages, success for confirmations, warning for cautions, and destructive for critical alerts.

@for (variant of variants; track variant) {

{{ meta[variant].title }}

{{ meta[variant].description }}

}

Without description

Omit ngAtomsAlertDescription for compact, single-line alerts. The layout adjusts automatically to center-align the icon.

@for (variant of variants; track variant) {

{{ meta[variant].title }}

}

API

Inputs on ngAtomsAlert. The component also exposes ngAtomsAlertTitle and ngAtomsAlertDescription directives for internal slots.

Prop Type Default Description
@for (prop of props; track prop.name) {
{{ prop.name }} {{ prop.type }} {{ prop.defaultVal }} {{ prop.description }}
}

Accessibility

Alerts are presentational by default. For live, dynamic alerts that appear after a user action, add role="alert" or aria-live="polite".
